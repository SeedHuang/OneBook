# 任务：需求提取

请从以下文档中提取结构化信息。

# 分析维度

1. 功能点识别 — 从文档中逐个提取独立功能点，确保不遗漏
2. 角色权限梳理 — 识别系统涉及的角色，构建权限矩阵
3. 业务流程还原 — 梳理关键流程的步骤、决策点、跳转路径
4. 数据实体定义 — 识别核心数据字段、计算规则、时效性
5. 风险预判 — 标注未决策事项和技术风险

# 输出模板

## 一、功能点列表

| 功能点ID | 所属模块 | 功能名称 | 实现端 | 关联角色 | 核心业务规则 | 数据来源 | 交互说明 | 设计稿参照 | 设计稿还原难度 |
|---------|---------|---------|--------|---------|------------|---------|---------|----------|-------------|

每个功能点必须有独立编号（如 HOM-001、WRK-001），可被后续任务拆分直接引用。
实现端标注：🟢[前端] / 🔵[后端] / 🟣[前后端]
- 设计稿参照：`✅ 设计稿有对应UI` / `❌ 设计稿未覆盖`（无设计稿时填 `-`）
- 设计稿还原难度：轻量 / 中等 / 重度（无设计稿时填 `-`）
